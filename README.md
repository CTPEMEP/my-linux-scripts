# My Linux Scripts Collection

Коллекция полезных bash-скриптов для администрирования Linux-систем, созданная в процессе изучения DevOps.

# О проекте

Этот репозиторий содержит набор bash-скриптов для повседневного администрирования Linux-серверов. Каждый скрипт решает конкретную задачу, имеет подробные комментарии и безопасен в использовании.

**Цели проекта:**
- Создать полезные инструменты для системного администратора
- Практиковать навыки bash-программирования
- Документировать код для обучения других
- Построить портфолио DevOps-инженера

# Скрипты

## 1. system-info.sh

**Назначение:**
Вывод комплексной информации о системе в удобочитаемом формате.

**Что показывает:**
- Информация об операционной системе (дистрибутив, версия)
- Версию ядра Linux
- Время непрерывной работы системы (uptime)
- Использование дискового пространства
- Статистику использования оперативной памяти
- Модель процессора
- Сетевые интерфейсы и их IP-адреса

**Использование:**

Базовое использование

`./system-info.sh`

Cохранение вывода в файл

`./system-info.sh > system_report.txt`

**Пример вывода:**
```
======================================
ИНФОРМАЦИЯ О СИСТЕМЕ
======================================
Операционная система:
AlmaLinux 9.3 (Shamrock Pampas Cat)

Версия ядра:
5.14.0-362.8.1.el9_3.x86_64

Время работы системы:
up 2 hours, 30 minutes
...
```

## 2. log-cleaner.sh

**Назначение:** Безопасная очистка старых log-файлов.

**Особенности:**

Безопасный по умолчанию — только показывает файлы, не удаляет

Настраиваемый период (по умолчанию 7 дней)

Поддержка разных директорий с логами

Показывает размер и дату изменения каждого файла

**Использование:**
```
# Просмотр логов старше 7 дней (без удаления)
./log-cleaner.sh

# Просмотр логов старше 30 дней
./log-cleaner.sh 30

# Указание другой папки с логами
./log-cleaner.sh 14 /var/log/nginx

# Реальное удаление (после настройки)
sudo ./log-cleaner.sh 30 /var/log
Настройка для реального удаления:

Откройте скрипт в редакторе:

nano log-cleaner.sh
Найдите строку: # rm "$file"

Удалите # в начале строки

Сохраните файл

Внимание: Будьте осторожны при удалении логов. Рекомендуется сначала протестировать в течение нескольких дней в режиме просмотра.
```
## 3. backup-configs.sh

**Назначение:** Автоматическое резервное копирование важных конфигурационных файлов.

**Что бэкапит:**

~/.bashrc, ~/.bash_profile — настройки оболочки Bash

~/.ssh/config — конфигурация SSH-клиента

/etc/ssh/sshd_config — настройки SSH-сервера

/etc/hosts — локальная таблица hosts

/etc/resolv.conf — DNS-настройки

**Особенности:**

Автоматическое именование с timestamp

Отчёт о размере архива

Проверка существования файлов перед бэкапом

Просмотр истории бэкапов

**Использование:**
```
# Создать бэкап
./backup-configs.sh

# Проверить созданные бэкапы
ls -la ~/config-backups/
```

**Структура бэкапов:**

```
~/config-backups/
├── config-backup-20240316_143022.tar.gz
├── config-backup-20240315_090011.tar.gz
└── config-backup-20240314_101545.tar.gz
```

# Установка и использование

**Требования**

ОС: Любой дистрибутив Linux (тестировалось на AlmaLinux 10.1)

Bash: версия 4.0 или выше

Утилиты: grep, awk, sed, find, tar, df, free, uptime

## Установка
```
# Клонировать репозиторий
git clone https://github.com/ваш-логин/my-linux-scripts.git
cd my-linux-scripts

# Дать права на выполнение
chmod +x *.sh

# Проверить установку
./system-info.sh
```
## Настройка (опционально)
Добавьте папку со скриптами в PATH для глобального доступа:

```
# Добавить в ~/.bashrc
echo 'export PATH="$PATH:$HOME/my-linux-scripts"' >> ~/.bashrc
source ~/.bashrc
```
Теперь можно запускать из любой директории

```system-info.sh```

# Тестирование

В проекте есть тестовый скрипт для проверки работы:

```
# Запустить все тесты
./test-scripts.sh

# Проверить синтаксис всех скриптов
for script in *.sh; do bash -n "$script" && echo "$script: OK"; done
```
